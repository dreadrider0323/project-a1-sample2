# スロット管理機能 要件定義（管理者）

## 機能概要

管理者が予約枠（スロット）を作成・編集・削除する機能を提供する。
スロットはカレンダー形式で表示され、利用者が予約申請する対象となる。

---

## EARS 要件

### スロット一覧表示

REQ-SLOT-001
: adminユーザーが「GET /admin/slots」にアクセスしたとき、システムは全スロットを月カレンダー形式で表示する。

REQ-SLOT-002
: システムは、カレンダー上の各スロットを対応する日付に開始時刻・終了時刻・場所とともに表示する。

REQ-SLOT-003
: システムは、adminユーザーがカレンダービューで月を移動できるようにする。

REQ-SLOT-004
: システムは、カレンダービューでアクティブなスロットと非アクティブなスロットを視覚的に区別する。

### スロット作成

REQ-SLOT-005
: adminユーザーが「GET /admin/slots/new」にアクセスしたとき、システムは日付・開始時刻・終了時刻・場所の入力フィールドを持つスロット作成フォームを表示する。

REQ-SLOT-006
: adminユーザーが有効なスロット作成フォームを「POST /admin/slots/new」で送信したとき、システムはデータベースに新しいスロットレコードを作成し、スロット一覧ページへリダイレクトする。

REQ-SLOT-007
: adminユーザーが「POST /admin/slots/new」で終了時刻が開始時刻以前のスロット作成フォームを送信したとき、システムはリクエストを拒否してバリデーションエラーを表示する。

REQ-SLOT-008
: adminユーザーが「POST /admin/slots/new」で必須フィールドが欠落したスロット作成フォームを送信したとき、システムはリクエストを拒否してバリデーションエラーを表示する。

REQ-SLOT-009
: adminユーザーが「POST /admin/slots/new」でCSRFトークンなしまたは無効なCSRFトークンのフォームを送信したとき、システムはリクエストを403レスポンスで拒否する。

REQ-SLOT-010
: システムは、新規作成されたスロットをデフォルトでアクティブ状態に設定する。

### スロット編集

REQ-SLOT-011
: adminユーザーが「GET /admin/slots/{slot_id}/edit」にアクセスしたとき、システムは既存のスロットデータが事前入力された編集フォームを表示する。

REQ-SLOT-012
: adminユーザーが有効なスロット編集フォームを「POST /admin/slots/{slot_id}/edit」で送信したとき、システムはデータベースのスロットレコードを更新し、スロット一覧ページへリダイレクトする。

REQ-SLOT-013
: adminユーザーが「POST /admin/slots/{slot_id}/edit」で終了時刻が開始時刻以前のスロット編集フォームを送信したとき、システムはリクエストを拒否してバリデーションエラーを表示する。

REQ-SLOT-014
: adminユーザーが存在しないスロットIDで「GET /admin/slots/{slot_id}/edit」にアクセスしたとき、システムは404レスポンスを返す。

REQ-SLOT-015
: adminユーザーが「POST /admin/slots/{slot_id}/edit」でCSRFトークンなしまたは無効なCSRFトークンのフォームを送信したとき、システムはリクエストを403レスポンスで拒否する。

REQ-SLOT-016
: システムは、adminが編集フォームからスロットのアクティブ状態を切り替えられるようにする。

### スロット削除

REQ-SLOT-017
: adminユーザーが「POST /admin/slots/{slot_id}/delete」で削除リクエストを送信したとき、システムは指定されたスロットと関連するすべての予約をデータベースから削除する。

REQ-SLOT-018
: adminユーザーが「POST /admin/slots/{slot_id}/delete」でCSRFトークンなしまたは無効なCSRFトークンの削除リクエストを送信したとき、システムはリクエストを403レスポンスで拒否する。

REQ-SLOT-019
: adminユーザーが存在しないスロットIDで「POST /admin/slots/{slot_id}/delete」に削除リクエストを送信したとき、システムは404レスポンスを返す。

REQ-SLOT-020
: スロットが削除されたとき、システムはそのスロットに関連するすべての予約レコードも削除する。

---

## 関連エンドポイント

| メソッド | パス | 説明 |
|---------|------|------|
| GET | `/admin/slots` | スロット一覧（月カレンダー） |
| GET | `/admin/slots/new` | スロット作成フォーム |
| POST | `/admin/slots/new` | スロット作成処理 |
| GET | `/admin/slots/{id}/edit` | スロット編集フォーム |
| POST | `/admin/slots/{id}/edit` | スロット更新処理 |
| POST | `/admin/slots/{id}/delete` | スロット削除処理 |

---

## データ仕様

### available_slots テーブル

| カラム | 型 | 必須 | 説明 |
|--------|-----|------|------|
| id | INTEGER | ○ | 主キー（自動採番） |
| date | DATE | ○ | 予約枠の日付 |
| start_time | TIME | ○ | 開始時刻 |
| end_time | TIME | ○ | 終了時刻（start_time より後） |
| location | TEXT | ○ | 場所・住所 |
| is_active | BOOLEAN | ○ | 有効フラグ（デフォルト: true） |
