# スケジュール管理システム 全体要件概要

## システム概要

本システムは、スケジュールを一元管理するWebアプリケーションである。
管理者・利用者・閲覧専用の3種類のロールを持つユーザーが協調して予約を管理する。

---

## ロール定義

| ロール | 識別子 | 概要 |
|--------|--------|------|
| 管理者 | admin | スロット作成・予約承認・ユーザー管理を行う |
| 利用者 | user | スロットを閲覧し予約申請を行う |
| 閲覧専用 | viewer | 申請状況の確認のみ行う |

---

## EARS 要件一覧（全体）

### システム基本動作

REQ-SYS-001
: システムは、Webベースのスケジュール管理アプリケーションを提供する。

REQ-SYS-002
: システムは、管理者（admin）・一般利用者（user）・閲覧専用（viewer）の3種類のユーザーロールをサポートする。

REQ-SYS-003
: ユーザーがルートURL「/」にアクセスしたとき、システムは認証済みセッションに基づいてロールに応じたトップページへリダイレクトする。

REQ-SYS-004
: アクティブなセッションなしでユーザーがルートURL「/」にアクセスしたとき、システムはログインページへリダイレクトする。

REQ-SYS-005
: システムは、認証不要で「GET /health」に対してヘルスチェックレスポンスを返す。

REQ-SYS-006
: システムは、全データをPostgreSQLデータベースに永続化する。

REQ-SYS-007
: システムは、Jinja2テンプレートを使用してサーバーサイドレンダリングされたHTMLとしてフロントエンドを提供する。

## システム制約

REQ-SYS-CON-001
: システムは、Google Cloud RunまたはFly.io上でDockerコンテナとしてデプロイできる。

REQ-SYS-CON-002
: システムは、実行時設定として環境変数「SECRET_KEY」と「DATABASE_URL」を使用して動作する。

REQ-SYS-CON-003
: 本番環境で動作中、システムはHSTSを含むHTTPS専用のセキュリティヘッダーを適用する。

REQ-SYS-CON-004
: システムは、外部CDNに依存せず、ローカルファイルストレージからすべての静的アセット（CSS・JavaScript・地図ライブラリ）を提供する。

## データモデル概要

### users テーブル
- ユーザー識別情報・ロール・ハッシュ化パスワードを保持する

### available_slots テーブル
- 管理者が作成した予約枠（日付・時間帯・場所）を保持する

### reservations テーブル
- 利用者の予約申請とそのステータス（pending / confirmed / rejected）を保持する

---

## 用語定義

| 用語 | 定義 |
|------|------|
| スロット | 管理者が設定した予約可能な時間枠 |
| 予約申請 | 利用者がスロットに対して申請した訪問予定 |
| pending | 予約申請後・承認前の状態 |
| confirmed | 管理者が承認した予約状態 |
| rejected | 管理者が却下した予約状態 |
