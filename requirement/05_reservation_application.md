# 予約申請機能 要件定義（利用者）

## 機能概要

利用者が管理者の作成したスロットを閲覧し、希望の時間帯で予約申請を行う機能を提供する。
申請フォームには地図ピッカーが搭載されており、訪問場所の位置確認が可能。

---

## EARS 要件

### スロット一覧表示

REQ-RESV-USER-001
: 利用者が「GET /user/slots」にアクセスしたとき、システムはアクティブな全スロットを月カレンダー形式で表示する。

REQ-RESV-USER-002
: システムは、各スロットを対応する日付に開始時刻・終了時刻・場所とともに表示する。

REQ-RESV-USER-003
: システムは、利用者がカレンダービューで月を移動できるようにする。

REQ-RESV-USER-004
: システムは、利用者向けカレンダーに「is_active = true」のスロットのみを表示する。

### 予約申請フォーム

REQ-RESV-USER-005
: 利用者が「GET /user/reserve/{slot_id}」にアクセスしたとき、システムは指定されたスロットの予約申請フォームを表示する。

REQ-RESV-USER-006
: 予約申請フォームは、スロットの日付・開始時刻・終了時刻・場所を表示する。

REQ-RESV-USER-007
: 予約申請フォームは、スロットの時間範囲内で希望開始時刻と希望終了時刻の入力フィールドを提供する。

REQ-RESV-USER-008
: 予約申請フォームは、スロットの場所を示すためにOpenStreetMapを使用したLeaflet.jsのインタラクティブマップを表示する。

REQ-RESV-USER-009
: 利用者が存在しないまたは非アクティブなスロットIDで「GET /user/reserve/{slot_id}」にアクセスしたとき、システムは404レスポンスを返す。

### 予約申請送信

REQ-RESV-USER-010
: 利用者が有効な予約フォームを「POST /user/reserve/{slot_id}」で送信したとき、システムは「status = pending」で新しい予約レコードを作成し、利用者の予約一覧ページへリダイレクトする。

REQ-RESV-USER-011
: 利用者が「POST /user/reserve/{slot_id}」でスロットの時間範囲外の希望開始時刻または希望終了時刻を含む予約フォームを送信したとき、システムはリクエストを拒否してバリデーションエラーを表示する。

REQ-RESV-USER-012
: 利用者が「POST /user/reserve/{slot_id}」でISO 8601形式以外の希望開始時刻を含む予約フォームを送信したとき、システムはリクエストを拒否してバリデーションエラーを表示する。

REQ-RESV-USER-013
: 利用者が「POST /user/reserve/{slot_id}」で希望終了時刻が希望開始時刻以前の予約フォームを送信したとき、システムはリクエストを拒否してバリデーションエラーを表示する。

REQ-RESV-USER-014
: 利用者が「POST /user/reserve/{slot_id}」で同一スロットへの予約が既に存在する状態で予約フォームを送信したとき、システムはリクエストを拒否して重複予約エラーを表示する。

REQ-RESV-USER-015
: 利用者が「POST /user/reserve/{slot_id}」でCSRFトークンなしまたは無効なCSRFトークンの予約フォームを送信したとき、システムはリクエストを403レスポンスで拒否する。

### マイ予約一覧

REQ-RESV-USER-016
: 利用者が「GET /user/reservations」にアクセスしたとき、システムは現在認証済みの利用者が申請した全予約を表示する。

REQ-RESV-USER-017
: システムは、一覧に各予約の対象スロット日付・希望開始時刻・希望終了時刻・現在のステータスを表示する。

REQ-RESV-USER-018
: システムは、利用者の予約一覧でpending・confirmed・rejectedの予約を視覚的に区別する。

## 関連エンドポイント

| メソッド | パス | 説明 |
|---------|------|------|
| GET | `/user/slots` | 予約可能スロット一覧（月カレンダー） |
| GET | `/user/reserve/{slot_id}` | 予約申請フォーム |
| POST | `/user/reserve/{slot_id}` | 予約申請処理 |
| GET | `/user/reservations` | マイ予約一覧 |

---

## バリデーション仕様

| 検証項目 | 条件 | エラー内容 |
|---------|------|-----------|
| 希望開始日時 | ISO 8601形式であること | フォーマットエラー |
| 希望終了日時 | ISO 8601形式であること | フォーマットエラー |
| 時間範囲 | スロットの開始〜終了時刻の範囲内であること | 範囲外エラー |
| 時間順序 | 希望終了日時 > 希望開始日時であること | 順序エラー |
| 重複申請 | 同一スロットへの重複申請がないこと | 重複エラー |

---

## 地図機能仕様

REQ-RESV-USER-MAP-001
: システムは、予約フォームページにLeaflet.jsを使用してスロットの場所を視覚化するインタラクティブマップを描画する。

REQ-RESV-USER-MAP-002
: システムは、地図の背景にOpenStreetMapのタイルデータを使用する。

REQ-RESV-USER-MAP-003
: システムは、外部CDNに依存せず、ローカル静的ファイルからLeaflet.jsとそのCSSを提供する。
